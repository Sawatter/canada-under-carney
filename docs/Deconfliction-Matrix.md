# Cross-Dimension Deconfliction Matrix

**Purpose:** Every major metric, event, or policy change has one primary scoring home. Secondary dimensions may reference it as context but cannot use it as grade-moving evidence. This prevents double-counting.

---

## How to Read This Matrix

- **Primary home:** This dimension grades it. Changes to this metric can move this dimension's grade.
- **Secondary mention:** These dimensions may reference it in rationale or context. It cannot move their grades.
- **Blocked:** This metric must not appear in these dimensions at all.

## Rules

1. **Primary home gets the grade impact.** If Food CPI changes, only Affordability Response's grade can move. Other dimensions may note it but cannot use it to justify a grade change.

2. **Secondary mentions must be labeled.** When a metric appears in a secondary dimension's rationale, it must be prefaced with context language: "As noted in [primary dimension]..." or "For reference..." It cannot appear in the "Key Metrics" section of a secondary dimension.

3. **If a metric appears in two dimensions' Key Metrics sections, one must be removed.** This is the mechanical test for double-counting.

4. **New metrics or events not on this matrix must be assigned a primary home before they can be used in scoring.** The monthly update process must include a deconfliction check.

5. **Flagship Delivery is a special case.** It draws on evidence from 5 flagship files (defence, housing, climate, immigration, major projects) but grades cross-cutting delivery capacity, not the policy merits of each file. Evidence from flagship files appears in Flagship Delivery as delivery indicators (did the government execute?), not as outcome indicators (is the policy good?). The distinction: housing starts appearing in Flagship Delivery grades whether the government is delivering its housing machinery, not whether housing is affordable.
