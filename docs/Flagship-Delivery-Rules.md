# Flagship Delivery Combination Rule

**Purpose:** Define how the 5 flagship files combine into a single Flagship Delivery grade. Eliminate analyst judgment in the combination step.

---

## The 5 Flagship Files

| # | File | What it measures | Primary dimension |
|---|---|---|---|
| 1 | **Defence** | NATO spending achieved, procurement milestones met | Defence & Trade |
| 2 | **Housing** | BCH pipeline, starts trajectory, federal spending execution | Housing Supply |
| 3 | **Major Projects** | MPO operational effectiveness, national interest designation use | Major Projects |
| 4 | **Climate** | Implementation of climate policy framework, ECCC capacity | Climate & Environment |
| 5 | **Immigration** | Levels plan execution, temp resident reduction, processing | Immigration |

**Important:** Flagship Delivery grades **delivery capacity** (is the government executing?), not **policy merit** (is the policy good?). A file can be "delivering" even if the policy it delivers is controversial.

---

## File Status Categories

Each flagship file is assessed as one of:

| Status | Definition |
|---|---|
| **Delivering** | Measurable progress against stated targets. Independent verification available. Execution on or near schedule. |
| **Partially delivering** | Some concrete action taken but pace or scale is significantly below targets. Mixed evidence. |
| **Stalled** | Announced but no measurable progress. Or: progress has stopped after initial action. |
| **Failing** | Measurable deterioration. Metrics moving in wrong direction. Active delivery breakdown. |

---

## Combination Rule

The Flagship Delivery grade is determined by the distribution of file statuses:

| Distribution | Grade | Logic |
|---|---|---|
| 4-5 delivering | **B+ to A-** | Broad-based execution capacity demonstrated |
| 3 delivering, 1-2 partial/stalled | **B- to C+** | Majority executing, some gaps |
| 2 delivering, 3 partial/stalled | **C** | Split record, execution selective |
| 1-2 delivering, 3-4 stalled | **C-** | Execution concentrated, most files not moving |
| 1 delivering, 1+ failing, rest stalled | **D+** | One bright spot, systemic delivery problems |
| 0-1 delivering, 2+ failing | **D to D-** | Broad delivery failure |
| 0 delivering, 3+ failing | **F** | Complete delivery collapse |

## One-Notch Move Rules

**Upgrade (+1 notch) requires:**
- At least one flagship file moves up one status category (e.g., stalled → partially delivering)
- AND no other file moves down
- OR: a previously stalled file shows first measurable delivery evidence

**Downgrade (-1 notch) requires:**
- At least one flagship file moves down one status category
- OR: a previously delivering file shows measurable deterioration

**Multi-notch move (+2 or -2) requires:**
- Two or more flagship files move status in the same direction simultaneously
- AND the release log explains why a single-notch move is insufficient
- This must be approved by the Referee lane in QA

---

## Overlap Prevention

Flagship Delivery uses evidence from the 5 flagship files but does NOT duplicate their grades. The distinction:

| Question | Answered by | NOT answered by |
|---|---|---|
| Is housing policy good? | Housing Supply | Flagship Delivery |
| Is the government executing on housing? | Flagship Delivery | Housing Supply |
| Are housing starts improving? | Housing Supply (primary metric) | Flagship Delivery (context only) |
| Is BCH disbursing funds on schedule? | Flagship Delivery (execution metric) | Housing Supply (context only) |

**The test:** If you removed Flagship Delivery entirely, would the information be lost? If yes, the metric belongs in Flagship Delivery. If the information already appears in another dimension's grade, it does NOT belong in Flagship Delivery.

---

## Probationary Status

Flagship Delivery is on **probation** through the May 2026 cycle. It will be retained if:
1. The combination rule produces grades that do not contradict the individual dimension grades
2. The deconfliction matrix successfully prevents double-counting
3. The dimension adds analytical value beyond what the individual dimensions already capture

If any of these conditions fail, Flagship Delivery becomes the leading candidate for removal in v2.0.
